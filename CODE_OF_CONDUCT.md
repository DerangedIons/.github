# Code of Conduct

This document sets norms for how we work together in the DerangedIons organization. It applies to everyone contributing to any DerangedIons repository, regardless of which organization employs them.

For the standard "be kind, don't harass, don't discriminate" baseline we follow the [Contributor Covenant](https://www.contributor-covenant.org/version/2/1/code_of_conduct/). The rest of this document covers the day-to-day mechanics that actually shape how this team operates.

## Work in the Open

Everyone communicates their work on GitHub through issues, pull requests (PRs), projects, and commit messages.  Acknowledge not all work can be tracked in this way.

Follow the PR and Issue templates.  Tell the story of *why*, not just *what*.  This is the primary mechanism in which the rest of team learns what you're doing and how it contributes to project goals.  What you write is historical record and should answer future questions e.g. "Why did we do X?", "Why did we not pursue Y?".

**Corollary:** If you want to know what someone is working on, read their PRs and issues before asking (or ask Claude to summarize it for you).  It is not the responsibility of authors to re-narrate on demand what they've already spent time documenting.

## Communication

- **Public over private.** Default to GitHub/Slack project channel, not DMs. DM only when the content is genuinely relevant only to the two of you. If you find yourself starting a DM about the science or the code, stop and post it in the channel.
- **Async-first.** We work across many timezones.  Prioritize asynchronous communication: write clearly, no stream-of-consciousness walls of text, etc.  A Slack `@mention` is a knock on their door, an interruption.  
  - *Corollary*: Use your Slack status to communicate your working state to the team, e.g. "Deep work until noon".  I.e. let people know if your office door is open or closed.
- **Weekly Update.** Post a paragraph in slack (can be Claude/AI-generated) of weekly progress, submitted ≥2 hours before the weekly lab meeting.

## Synchronous Cadence

As a distributed team, we are asynchronous by default.  Synchronous communication is less frequent, but arguably more important.  Meetings require purpose and a goal.  

- **Weekly lab meeting** (60 min, mandatory) with default agenda:
  1. Rapid round of updates (≤ 2 min/person) describing nuances around weekly updates in Slack.
  2. Blockers
  3. Decisions
- **Subproject syncs** (~30 min, as needed) for active blockers or cross-thread questions that require discussion.

## Blockers

We are working on hard unsolved problems.  It is the expectation that everyone will hit roadblocks and need to communicate it with the team.

- **No silent struggling.** Stuck for more than a day? Say so, in writing, in public.  Write a detailed GitHub Issue of the problem, then post a link in Slack.

## AI Usage

- **Humans are authors.** E.g. Claude is *never* the author of a commit.  Humans are responsible for the work and should be attributed.
